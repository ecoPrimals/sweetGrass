## 🌟 Key Insights

### 1. Semantic > Syntactic ⭐
**Git**: Line counts (unfair)  
**SweetGrass**: Semantic weights (fair!)

Example:
- Git: Alice 60% (boilerplate), Bob 40% (algorithm)
- SweetGrass: Alice 40% (structure), Bob 60% (core value)

### 2. Fair Attribution
Weight by **contribution type**:
- Creation: 1.0
- Implementation: 0.8
- Optimization: 0.6
- Refactoring: 0.4
- Bug fix: 0.3

### 3. Provable
- Cryptographic signatures (BearDog)
- Merkle inclusion proofs (rhizoCrypt)
- Immutable history (LoamSpine)
- **Legally admissible!**

### 4. Query-able
Ask semantic questions:
- "Who contributed to OAuth feature?"
- "What did Alice create?"
- "How did module X evolve?"
