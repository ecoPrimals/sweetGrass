## 📝 Documentation Standards

### Principles
- ✅ Clear, concise, accurate
- ✅ Code examples for all patterns
- ✅ Real execution (no mocks in showcase)
- ✅ Progressive learning paths
- ✅ Comprehensive but not overwhelming

- `README.md` - As features change
- `reports/` - After audits/milestones

## 🌾 Philosophy

All SweetGrass documentation follows these principles:

1. **"Interactions show us gaps in our evolution"**
   - Real testing reveals real gaps
   - Gaps are documented, not hidden
   - Learning from real execution

2. **"No mocks in showcase"**
   - All demos use real binaries
   - Real services, real RPC
   - Zero mocks = real learning

3. **"Every piece of data has a story"**
   - Complete provenance tracking
   - Fair attribution for all
   - Transparent, auditable

4. **"Primal sovereignty"**
   - Capability-based discovery
   - No hardcoded dependencies
   - Runtime configuration
